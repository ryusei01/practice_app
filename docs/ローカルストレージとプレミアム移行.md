# ローカルストレージとプレミアム移行

端末の AsyncStorage に保存されるデータの整理と、`POST /api/v1/answers/migrate-local-data`（クラウド移行）の対象範囲をまとめたものです。有料プラン関連の運用チェックは [TODO_有料プラン.md](./TODO_有料プラン.md) と併せて参照してください。

---

## 1. プレミアム移行（migrate-local-data）の概要

| 項目 | 内容 |
|------|------|
| エンドポイント | `POST /api/v1/answers/migrate-local-data`（`backend/app/api/answers.py`） |
| 認可 | ログイン必須。**`is_premium` が true でないと 403** |
| フロントの入口 | `frontend/app/(app)/premium-upgrade.tsx` の「データを移行する」（プレミアム時のみ表示） |
| 成功後 | `answers_${userId}` と `question_sets_${userId}` を **削除**（移行済みとしてローカル掃除） |

移行リクエストのペイロードは `answers` と `question_sets` の配列。サーバー側では回答の重複（同一 `answered_at`）や問題集の重複（同一ユーザーの **タイトル＋カテゴリ**）を避けて挿入し、回答が増えた場合は統計再計算（`StatsUpdater.recalculate_all_stats`）が走る。

---

## 2. 移行対象として読み込むキー（フロント）

| キー | 読み取り |
|------|----------|
| `answers_${user.id}` | `premium-upgrade.tsx` |
| `question_sets_${user.id}` | 同上 |

**書き込み元（現状のコードベース）**

- **`answers_${user.id}`**  
  - **無料かつログイン**のとき、通常の **クイズ**（`frontend/app/(app)/quiz/[id].tsx`）完了時に、回答配列へ追記する。  
  - **プレミアム**のクイズではこのキーには積まず、API（`answersApi.submitAnswer`）へ直接保存する想定。

- **`question_sets_${user.id}`**  
  - 移行処理はこのキーを読むが、**このキーへ `setItem` している箇所はリポジトリ内では見つかっていない**（将来用・未接続の可能性）。  
  - ログイン後の「問題集作成」は API 経由（`question-sets/create.tsx`）のため、自作データがこのキーに溜まらない構成になっている。

---

## 3. 移行の対象外（代表例）

以下は **migrate-local-data では送られない**／**別用途**です。

| キーまたはプレフィックス | 役割 |
|--------------------------|------|
| `@flashcard_answers_${問題集ID}` | フラッシュカードの**問題集単位**の回答履歴（追記）。詳細は後述。 |
| `@srs_state_${問題集ID}` | SRS（間隔反復）の状態。`srsService` が読み書き。 |
| `downloaded_question_sets_${user.id}` | マイ問題集から「ダウンロード」した問題集＋全問のローカルキャッシュ。保存のみで、移行対象外。 |
| `@trial_question_sets` など | 未ログイン・トライアル用（`localStorageService`）。 |
| `@quiz_progress_${問題集ID}` | クイズの途中再開用。 |
| `@redsheet_progress_*` / `@redsheet_mode_*` | 赤シート等の UI 状態（`localStorageService`）。 |
| `study_records_v1` | 学習記録（`studyRecordService`）。 |
| `pref_showDefaultSets` / `app_language` など | UI・設定。 |

## 4. `@flashcard_answers_${問題集ID}` の詳細

**定数:** `FLASHCARD_KEY_PREFIX = "@flashcard_answers_"`（`frontend/src/services/srsService.ts`）

### 4.1 保存されるデータ

フラッシュカード画面（例: `frontend/app/(app)/flashcard/[id].tsx`）でセッション完了時 `submitAllAnswers` が実行され、**既存配列に追記**される。要素の例:

- `question_id`
- `question_set_id`
- `is_correct`, `answer_time_sec`
- `answered_at`（ISO 8601 文字列）

古い履歴の整理として `localStorageService.cleanupOldAnswers`（問題集 ID 単位・最大件数）が呼ばれる実装がある。

### 4.2 主な利用箇所

1. **問題集詳細**（`frontend/app/(app)/question-sets/[id].tsx`）  
   - `loadAnswerStats` で読み込み、問題ごとの試行回数・正答率・最終回答日などを集計して表示。

2. **`srsService`**（`frontend/src/services/srsService.ts`）  
   - **`getMostIncorrectQuestions`**: 上記キーから間違い回数を集計し、優先出題用に利用。  
   - **`initializeFromHistory`**: SRS マップが空のとき、フラッシュカード履歴から SM-2 風の初期状態を構築して `@srs_state_${setId}` に保存。

### 4.3 API との関係

ログイン済みの通常版では、フラッシュカード完了時に **`answersApi.submitAnswer` へも送信**する分岐がある（トライアルでない場合）。  
そのため「クラウドの回答テーブル」と「ローカルの `@flashcard_answers_*`」が**並行**して存在しうる。  
一方、**プレミアム移行**が読むのは `answers_${userId}` 側のため、**フラッシュカード専用キーだけ**をまとめて DB に送る処理は **含まれていない**。

## 5. `downloaded_question_sets_${user.id}` の詳細

**ファイル:** `frontend/app/(app)/my-question-sets/index.tsx` の `handleDownload`

- `questionSetsApi.download(questionSet.id)` で取得した **`QuestionSetWithQuestions`** を、ユーザーごとの配列にマージして保存（同一 `id` は上書き）。
- **用途:** 端末へのオフライン用の控え・再取得のためのキャッシュ想定。
- **現状:** このキーを **一覧表示や別画面で読み戻している箇所は同一ファイル内の保存処理以外には見つかっていない**（将来のオフライン対応の残り、などの可能性）。

## 6. クイズの回答保存の違い（整理）

| モード | 無料（ログイン） | プレミアム |
|--------|------------------|------------|
| **通常クイズ** | `answers_${userId}` に JSON 追記 | API へ `submit`（ローカル `answers_*` には積まない） |
| **フラッシュカード** | `@flashcard_answers_${setId}` に追記 ＋ ログイン時は API にも送信 | 同左（ローカルキーは共通）※ `submitAnswer` 側の課金制限は API 実装に依存 |

※「無料で API に送っているか」は `answers/submit` のサーバー仕様とフラッシュカード側の分岐の整合を別途確認するとよい。

## 7. 自動テスト（API 挙動）

`backend/tests/test_migrate_local_data.py` で以下を検証している:

- 非プレミアム → **403**
- プレミアム → 回答・問題集・問題の挿入件数と、同一 `answered_at` の **重複スキップ**

# 例（backend 仮想環境で）
