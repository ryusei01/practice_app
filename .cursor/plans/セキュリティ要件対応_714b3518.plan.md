# Google OAuth セキュリティ要件対応プラン

## 現状サマリー

### 既に実装済み（対応不要）

- **Google OAuth ログイン**: `expo-auth-session` + `/auth/google` エンドポイント
- **JWT管理**: アクセストークン30分 / リフレッシュトークン7日
- **リフレッシュトークンのハッシュ化保存**: Argon2でDB保存
- **セキュリティヘッダー**: X-Frame-Options, X-Content-Type-Options, HSTS(本番のみ)
- **パスワード平文保存なし**: Google OAuthユーザーは`hashed_password = null`

---

## 対応が必要な項目

### 必須（法的要件）

#### 1. プライバシーポリシーページ（**未実装**）

- 専用ページが存在しない（register.tsxのモーダル内に断片的な記述があるのみ）
- 個人情報保護法上、独立したページとして公開が必要
- 記載内容: 取得情報（Googleアカウントのメール・名前・google_id）、利用目的、第三者提供なし、保存期間、削除方法
- 追加ファイル: `frontend/app/(public)/privacy-policy.tsx`（またはWebビュー用の静的ページ）

#### 2. ログイン試行制限（**未適用**）

- `slowapi`はインストール済みで`main.py`に登録されているが、`/auth/google`エンドポイントに`@limiter.limit`デコレータが未適用
- `[backend/app/api/auth.py](backend/app/api/auth.py)` の`google_login`関数に追加が必要

```python
# 現状（制限なし）
@router.post("/google", response_model=TokenResponse)
async def google_login(request: GoogleAuthRequest, db: Session = Depends(get_db)):

# 修正後
@router.post("/google", response_model=TokenResponse)
@limiter.limit("10/minute")
async def google_login(request: Request, body: GoogleAuthRequest, db: Session = Depends(get_db)):
```

### セキュリティ強化（推奨）

#### 3. Googleトークンの`aud`クレーム検証（**未実装**）

- 現在はuserinfo APIへのアクセス成功のみで検証しており、`GOOGLE_CLIENT_ID`による`aud`検証なし
- 他アプリ向けのGoogleトークンを使ったなりすましが可能な状態
- `[backend/app/api/auth.py](backend/app/api/auth.py)` にて`google_data.get("aud")`を検証するロジック追加

#### 4. フロントエンドのトークン保存（**AsyncStorage → SecureStore**）

- `[frontend/src/api/auth.ts](frontend/src/api/auth.ts)` が`AsyncStorage`を使用（暗号化なし）
- `expo-secure-store`への移行が推奨
- モバイルのKeychainを使用した安全な保存に変更

#### 5. デバッグログの除去（**本番リスク**）

- `[backend/app/core/auth.py](backend/app/core/auth.py)` でトークンの先頭20文字とユーザーメールをprint出力中
- `settings.DEBUG`フラグで制御するか、本番では削除

#### 6. CSP・Referrer-Policyヘッダーの追加

- `backend/app/main.py`の`add_security_headers`ミドルウェアに追加

## 本番デプロイ前に必要な環境変数設定（コード変更なし）

- `SECRET_KEY`: 強力なランダム文字列に変更（現在デフォルト値が弱い）
- `CORS_ORIGINS`: 本番ドメインのみに限定
- `DEBUG=false`: セキュリティヘッダー(HSTS)の有効化
