# 復習開始を「SM-2」vs「忘却曲線」で選べるようにする

## 目的

- (trial) と (app) の「クイズを開始」→問題選択モーダル内で、復習開始方法を2択にする
  - **SM-2復習**: `nextReviewDate <= now`（期限切れ/到来）の問題だけ出題
  - **忘却曲線復習**: 保持率 `R(t)=exp(-t/S)` が低い順に **上位N問**を出題（Nは既存の「問題数」入力を流用）

## 変更対象

- (trial) セット詳細
  - `[frontend/app/(trial)/set/[id].tsx](frontend/app/(trial)`/set/[id].tsx)
- (app) 問題集詳細
  - `[frontend/app/(app)/question-sets/[id].tsx](frontend/app/(app)`/question-sets/[id].tsx)
- SRSサービス（ランキング機能追加）
  - `[frontend/src/services/srsService.ts](frontend/src/services/srsService.ts)`

## 仕様（モーダル内の選択肢）

モーダル下部の開始ボタン群を以下にする。

- **通常クイズ開始**: 既存の `handleStartQuizWithSelection()`（全問/AI/カテゴリ/ランダム等）
- **SM-2で復習開始**: `srsService.getDueQuestions(setId)` で抽出 → `questionIds` に詰めてクイズ画面へ遷移
- **忘却曲線で復習開始**: `srsService.getLowestRetentionQuestions(setId, N, questionIds)` で抽出 → 同様に遷移

※ N は既存の「問題数」入力（AI選出・ランダム選出で使っている入力値）を流用。

## 実装詳細

### 1) `srsService` に「保持率が低い順」の抽出APIを追加

`[frontend/src/services/srsService.ts](frontend/src/services/srsService.ts)`

- 追加メソッド:
  - `getLowestRetentionQuestions(setId: string, n: number, questionIds: string[]): Promise<string[]>`

動作:

- `initializeFromHistory` / `getSRSMap` で SRS 状態を取得
- `questionIds` のうち、状態がないものは「未学習」として保持率を最小扱い（=優先的に出す）
- `retention` を計算して昇順でソートし、上位 n 件の `questionId` を返す

### 2) (trial) モーダルに 2つの復習開始ボタンを追加

`[frontend/app/(trial)/set/[id].tsx](frontend/app/(trial)`/set/[id].tsx)

- 新規ハンドラ:
  - `handleStartSm2ReviewQuiz()` → `getDueQuestions` 使用
  - `handleStartForgettingCurveReviewQuiz()` → `getLowestRetentionQuestions(id, questionCount, questionSet.questions.map(q=>q.id))`
- どちらも `questionIds` クエリを付けて `/(trial)/quiz/${id}` に遷移
- 復習対象が 0 件ならモーダル/エラー表示（既存の `showErrorModal` を流用）

UI:

- 既存の `Start Quiz` ボタン（通常開始）の近くに、
  - `SM-2で復習開始（○問）`
  - `忘却曲線で復習開始（○問）`
  を追加

### 3) (app) モーダルに 2つの復習開始ボタンを追加

`[frontend/app/(app)/question-sets/[id].tsx](frontend/app/(app)`/question-sets/[id].tsx)

- (trial) と同じ2ハンドラを追加
- 遷移先は `/(app)/quiz/${id}?questionIds=...`
- エラー表示は `Alert.alert` or 既存モーダル helper のどちらかに合わせる（ファイル内の流儀に合わせる）

## 既存の「復習する」ボタンとの整合

- 画面下部に追加済みの「○問を復習する」ボタンは、デフォルトで **SM-2復習**（期限切れ）として維持
- ただし、ユーザーが必ず2択したい場合は、下部ボタンを「復習を選ぶ」→モーダルを開く形に変更も可能（別タスク）

## テスト観点（手動）

- 1度も解いてない問題セットで:
  - 忘却曲線復習 → 未学習が優先されて N 問出る
  - SM-2復習 → 初期状態では 0 件になる可能性（メッセージが出る）
- クイズ/フラッシュカードで数問解いた後:
  - SM-2復習 → 期限切れがあると抽出される
  - 忘却曲線復習 → 保持率の低いものが優先される
